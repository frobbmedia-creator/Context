Technical Deep Dive & FAQ

How does Context read my workspace without sending it to a server?
Browser version uses the File System Access API + workers. CLI version reads the filesystem directly. In both cases your code never leaves the machine.
How does the Git-aware prioritization work?
The CLI runs real git status --porcelain and git ls-files. Any working-tree change gets priority 100. The browser version uses a lighter index + mtime heuristic. Neither writes to your repository.
Is the CLI free?
The source is MIT-licensed. You can install and run it today with npm install -g @frobb-media/context. The $9/mo Pro tier supports continued development and unlocks priority features and support.
What is a Content Security Policy (CSP), and why is yours so strict?
A Content Security Policy is a security firewall enforced by the browser. Context keeps scripts and workers self-hosted, allows same-origin Vercel Analytics requests, and does not upload workspace file contents.
Can the application handle massive repositories?
Yes. Processing is non-blocking (workers in the browser, efficient streaming in the CLI). Ignore rules are applied early. Memory stays under control even on large monorepos.
